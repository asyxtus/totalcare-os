# schema.sql

This folder holds `schema.sql` — a snapshot of the database's current state
(functions, triggers, tables, enums), generated by `npm run snapshot`.

It does not exist until you run the snapshot for the first time:

```
npm run snapshot
```

Once generated, commit `schema.sql` to git. It's the single source of truth
for what's live — read it instead of hunting through migration history when
you need to know what a function currently does.
